# Task Time Tracker API

REST-сервис для учета рабочего времени сотрудников. Система позволяет создавать задачи, управлять их статусами и фиксировать временные интервалы, затраченные на выполнение конкретных задач.

## Технологический стек

* **Язык программирования:** Java 17+
* **Фреймворк:** Spring Boot 3.x (Web, Validation)
* **Персистентность:** MyBatis
* **База данных:** PostgreSQL (основная), H2 (тестовая)
* **Сборка:** Maven
* **Тестирование:** JUnit 5, Mockito, AssertJ
* **Дополнительно:** Lombok

## Архитектурные особенности

* **Слой DAO:** Реализован с использованием MyBatis мапперов.
* **Валидация:** Все входящие данные (DTO) проходят проверку через Bean Validation API (JSR-303).
* **Обработка ошибок:** Реализован централизованный обработчик исключений через `@RestControllerAdvice`, возвращающий унифицированные JSON-ответы об ошибках.
* **Бизнес-логика:** Вынесена в слой сервисов, включая проверки на существование сущностей и корректность временных интервалов.

## Сущности (Модели данных)

### Task (Задача)
Представляет собой объект работы, над которой трудятся сотрудники.

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| id | Long | Уникальный идентификатор |
| title | String | Заголовок задачи (обязательно) |
| description | String | Подробное описание |
| taskStatus | Enum | Статус: NEW, IN_PROGRESS, DONE |

### TimeRecord (Запись времени)
Фиксирует время, затраченное конкретным сотрудником на задачу.

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| id | Long | Уникальный идентификатор |
| employeeId | Long | Идентификатор сотрудника |
| taskId | Long | Идентификатор связанной задачи |
| startTime | LocalDateTime | Время начала работ |
| endTime | LocalDateTime | Время завершения работ |
| workDescription | String | Описание проделанной работы |

## Запуск приложения

### Требования
* JDK 17 или выше
* Maven 3.6+
* Docker и Docker Compose (для запуска PostgreSQL)

### Шаги запуска
1.  **Запуск базы данных:**
    ```bash
    docker-compose up -d
    ```
    Это запустит контейнер PostgreSQL и выполнит скрипт `init.sql` для создания необходимых таблиц.

2.  **Сборка проекта:**
    ```bash
    mvn clean package -DskipTests
    ```

3.  **Запуск сервиса:**
    ```bash
    mvn spring-boot:run
    ```
    Сервис будет доступен по адресу: `http://localhost:8080`

## API Эндпоинты

### Управление задачами

#### 1. Создание задачи
* **Метод:** `POST`
* **Путь:** `/api/tasks`
* **Пример запроса:**
```json
{
  "title": "Разработка мапперов",
  "description": "Создать интерфейсы MyBatis для работы с БД"
}
```

#### 2. Получение задачи по ID
* **Метод:** `GET`
* **Путь:** `/api/tasks/{id}`

#### 3. Изменение статуса
* **Метод:** `PATCH`
* **Путь:** `/api/tasks/{id}/status?status=IN_PROGRESS`

---

### Учет времени

#### 1. Создание записи времени
* **Метод:** `POST`
* **Путь:** `/api/time-records`
* **Пример запроса:**
```json
{
  "employeeId": 101,
  "taskId": 1,
  "startTime": "2026-05-02T09:00:00",
  "endTime": "2026-05-02T12:00:00",
  "workDescription": "Реализация методов в TaskMapper"
}
```

#### 2. Получение записей сотрудника за период
* **Метод:** `GET`
* **Путь:** `/api/time-records`
* **Параметры:** `employeeId`, `start`, `end` (формат ISO-8601)
* **Пример:**
  `GET /api/time-records?employeeId=101&start=2026-05-01T00:00:00&end=2026-05-31T23:59:59`

## Обработка исключений

В случае возникновения ошибки API возвращает объект следующей структуры:

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| status | int | HTTP статус код (например, 404, 400) |
| message | String | Описание ошибки |
| timestamp | String | Время возникновения ошибки |

**Типовые коды:**
* `400 Bad Request` — ошибка валидации входных данных или бизнес-логики.
* `404 Not Found` — ресурс (задача или запись) не найден в базе данных.
* `500 Internal Server Error` — непредвиденная ошибка сервера.
